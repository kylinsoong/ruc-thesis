=== 4.1 Logistic Regression（逻辑回归）

==== 模型简介

逻辑回归是一种经典的线性分类模型，通过 sigmoid 函数将线性回归的输出映射到 [0,1] 区间，表示样本属于正类的概率。

==== 适用场景

* 金融风控的基线模型
* 需要模型解释性的场景
* 特征与标签呈线性关系的任务

==== 运行命令

.运行 Logistic Regression 在 German Credit 数据集
[source,bash]
----
python run_baseline_experiment.py --model lr --dataset german_credit
----

.运行 Logistic Regression 在 Credit Card 数据集
[source,bash]
----
python run_baseline_experiment.py --model lr --dataset credit_card
----

==== 参数说明

|===
|参数 |说明 |默认值

|`--model`
|模型标识，固定为 `lr`
|必填

|`--dataset`
|数据集标识
|必填

|`--output`
|结果输出路径
|`results/baseline_results.csv`
|===

==== 预期输出示例

[source,text]
----
Running lr on german_credit
============================================================

  recall: 0.7333
  f1: 0.6069
  auc_roc: 0.7902
  auc_pr: 0.6073

Result saved to results/baseline_results.csv
----

==== 注意事项

* 逻辑回归对特征尺度敏感，建议预处理时进行标准化
* 使用 `class_weight='balanced'` 自动处理类别不平衡

'''

=== 4.2 Random Forest（随机森林）

==== 模型简介

随机森林是一种集成学习方法，通过构建多棵决策树并取其投票结果来做出最终预测，具有抗过拟合能力强、对缺失值鲁棒等优点。

==== 适用场景

* 特征维度较高的数据集
* 需要特征重要性分析的场景
* 处理类别不平衡问题

==== 运行命令

.运行 Random Forest 在 German Credit 数据集

python run_baseline_experiment.py --model rf --dataset german_credit

=== 4.6 LLM-only（大语言模型）

==== 模型简介

LLM-only 模型使用火山引擎豆包大模型对金融风控场景进行评估，通过 API 调用获取模型的推理能力。

==== 适用场景

* 需要大语言模型推理能力的场景
* 复杂风控规则的评估
* 需要自然语言解释的场景

==== 运行命令

WARNING: LLM 模型调用真实 API 会产生费用。建议先使用 Mock 模式测试。

.标准运行（调用真实 API）
[source,bash]
----
# German Credit 数据集
python run_baseline_experiment.py --model llm --dataset german_credit

# Credit Card 数据集
python run_baseline_experiment.py --model llm --dataset credit_card
----

.Mock 模式运行（不调用 API，用于测试）
[source,bash]
----
# German Credit 数据集
python run_baseline_experiment.py --model llm --dataset german_credit --mock-llm

# Credit Card 数据集
python run_baseline_experiment.py --model llm --dataset credit_card --mock-llm
----

==== API 配置说明

.确保 `.env` 文件配置正确
[source,bash]
----
ARK_API_KEY=your_api_key_here
----

== 5. 其他运行方式

=== 5.1 运行指定模型在所有数据集

使用 `--all-datasets` 参数可以运行指定模型在两个数据集上：

[source,bash]
----
python run_baseline_experiment.py --model lr --all-datasets
----

=== 5.2 重复实验

使用 `--n-repeats` 参数设置重复次数，会输出均值和标准差：

[source,bash]
----
# 运行 5 次重复实验
python run_baseline_experiment.py --model lr --dataset german_credit --n-repeats 5
----

重复实验输出示例：

[source,text]
----
--- Average over 5 repeats ---
  accuracy: 0.7150 ± 0.0100
  precision: 0.5180 ± 0.0150
  recall: 0.7330 ± 0.0200
  f1: 0.6070 ± 0.0120
  auc_roc: 0.7900 ± 0.0080
  auc_pr: 0.6070 ± 0.0100
----

=== 5.3 自定义输出路径

使用 `--output` 参数指定结果输出路径：

[source,bash]
----
python run_baseline_experiment.py --model lr --dataset german_credit --output results/my_results.csv
----

== 9. 常见问题

[qanda]
LLM 模型调用报错？::
+
--
. 检查 `.env` 文件中的 `ARK_API_KEY` 是否正确配置
. 确认 API 密钥有足够的调用额度
. 或使用 `--mock-llm` 参数进行测试
--

DeepFM/TabNet 训练很慢？::
+
--
. 深度学习模型训练时间较长是正常现象
. 可以通过减少 epoch 数来加速（修改代码中的 `epochs` 参数）
. 建议使用 GPU 加速训练
--

结果与预期差异大？::
+
--
. 检查数据是否正确加载
. 确认随机种子是否一致（使用 `--n-repeats 1` 单次运行）
. 查看完整输出日志排查问题
--

如何只查看某个模型的结果？::
+
--
[source,bash]
----
grep "lr," lab/results/baseline_results.csv
----
--

如何清除历史结果？::
+
--
[source,bash]
----
rm lab/results/baseline_results.csv
----
或重定向到新文件：
[source,bash]
----
python run_baseline_experiment.py --model lr --dataset german_credit --output results/new_results.csv
----
--
